공개키와 개인키

블록체인은 비대칭키 알고리즘을 사용하는데, 일반적으로 🔑 개인키로 암호화를 한 후 위변조 여부를 확인하기 위해 제공되는 공개키로 복호화를 하게 됨. 이 과정을 정상적으로 진행하려면 두 개의 키가 필요하며, 흔히 메타마스크 지갑 주소가 '공개키'임.

니모닉 단어

키를 발급하는 과정에서 🆕 특정한 단어를 대입하여 키 값을 생성하게 되는데, 이는 알고리즘이 노출되어도 유추하기 어렵게 만들기 위한 일종의 보호 장치라고 생각하면됨. 메타마스크를 가입할 때 주어지는 12개의 단어가 바로 그 역할을함. 특히 이 단어는 그 단어의 순서까지 중요하게 여겨지는데, 이 단어를 통하여 개인키가 만들어지기 때문.

      const mnemonic = lightwallet.keystore.generateRandomSeed();
      return res.json({ mnemonic });

      console.log(err);
    }

    const { password, mnemonic } = req.body;
    try {
      lightwallet.keystore.createVault(
        {
          password: password,
          seedPhrase: mnemonic,
          hdPathString: "m/0'/0'/0'",
        },
        function(err, ks){
          ks.keyFromPassword(password, function(err, pwDerivedKey) {
            if(err) throw err;
            ks.generateNewAddress(pwDerivedKey, 1);
            const address = ks.getAddresses().toString();
            const keystore = ks.serialize();
